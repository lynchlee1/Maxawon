# 현재 디자인 룰 요약

- 새 UI를 만들 때는 기존 입력 필드, 파일 선택 행, 체크박스, 버튼, 섹션 구조를 먼저 재사용하고, 같은 목적의 스타일을 새로 정의하지 않는다.
- 체크박스 항목은 피하고, 기본값이 보이는 Y/N 선택 컨트롤로 표현한다.
- 버튼은 `RUN`, `STOP`, `OPEN`, `SAVE`처럼 즉시 동작을 실행하는 명령에만 쓰고, 상태값을 바꾸는 용도로 버튼형 토글을 남용하지 않는다.
- 변경은 요청된 화면과 동작에 필요한 범위로 제한하고, 주변 레이아웃이나 문구는 깨지는 문제가 있을 때만 함께 고친다.
